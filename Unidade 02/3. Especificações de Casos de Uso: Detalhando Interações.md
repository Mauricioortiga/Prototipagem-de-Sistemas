Uma técnica poderosa e amplamente utilizada para documentar requisitos funcionais é a **especificação de casos de uso.**

🎈Criado por **Ivar Jacobson (1980)**, se tornou popularizada pela UML, essa técnica permite descrever como usuários (e outros sistemas) interagem com o sistema para atingir objetivos específicos.

Caso de uso representa uma sequência de interações entre um ator e o sistema que resulta em algo de valor para o ator.

🎈Page-Jones (2001) define:

*Caso de uso é a interação definida e intencional entre um sistema e um ser humano, ou um ator não humano, que está desempenhando um papel específico fora do sistema.*

**Anatomia de um caso de uso:**

A especificação completa de caso de uso inclui diversos elementos estruturados como:

Elementos básicos: 

- Nome do caso de uso
- Ator principal
- Objetivo
- Pré condições
- Pós-condições de sucesso

**Com base em uma estrutura detalhada, temos uma visão completa da interação, permitindo que desenvolvedores compreendam não apenas o fluxo feliz, mas também exceções e situações alternativas.

Cockburn (2025) destaca:

Casos de uso bem escritos são instrumentos essenciais de comunicação entre stakeholders e desenvolvedores, servindo como base para planejamento de testes e validação de usuários.
