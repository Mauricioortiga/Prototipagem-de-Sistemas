⚡**A documentação de requisitos representa o primeiro passo no processo de desenvolvimento de sistemas computacionais.**

**Consiste em registrar de forma estruturada e compreensível, todas as necessidades e expectativas que o sistema deve atender.**

Exemplo: Construir uma casa: Definimos quantos cômodos terá, onde ficarão as portas e janelas, qual acabamento será utilizado, etc.

O mesmo princípio se aplica ao desenvolvimento de software.

🎈Sommerville (2003) destaca:

*O documento de requisitos de sistema (chamado às vezes de especificação funcional) deve definir exatamente o que deve ser implementado. 
Pode fazer parte do contrato entre o adquirente do sistema e os desenvolvedores de software.*

⚡ A documentação eficaz de requisitos traz benefícios para todos os envolvidos no projeto, dentre elas:

1. Ambiguidades.

2. Facilitação na comunicação entre stakeholders técnicos e não técnicos.

3. Base para estimativas de prazo e custo.

4. Orientação de trabalho das equipes de desenvolvimento e teste.

5. Funciona como referência para futuras manutenções e evoluções do sistema.

🎈Cohn e Sabbagh(2022) descreve:

*O que os métodos ágeis defendem não é a ausência de documentação, mas sim documentação suficiente e no momento adequado.*

**A documentação deve agregar valor, se mantida atualizada e estar alinhada com a realidade do projeto.**

🔨 **Requisitos do negócio: Entendendo o contexto organizacional:**

Os requisitos do negócio representam as necessidades organizacionais de alto nível que justificam o investimento no desenvolvimento do sistema.

**Em uma pergunta clássica, séria: Por que este sistema precisa existir?**

⚡ **Os requisitos de negócio estão intimamente ligados aos objetivos estratégicos da organização.**

Exemplo: Uma empresa de varejo pode ter como objetivo - Aumentar as vendas online em 30% no próximo ano.

**Este objetivo de negócio pode montar requisitos como desenvolver um aplicativo móvel de vendas, implementar um sistema
de recomendação personalizada ou criar um programa de fidelidade digital.**

🎈Campos(2014) ressalta:

*Compreender os processos de negócio é fundamental para extrair requisitos que realmente agreguem valor.*

**A modelagem com BPMN pode auxiliar nesta compreensão, permitindo visualizar como o trabalho flui pela organização e onde o sistema computacional pode
otimizar operações, reduzir custos ou melhorar a experiência do cliente.**

⚡ **Para documentar requisitos de negócio de forma eficaz, precisamos analisar:**

1. Identificar os objetivos estratégicos da organização.

2. Compreender os processos de negócio existentes.

3. Mapear problemas e oportunidades atuais.

4. Definir métricas de sucesso mensuráveis.

5. Estabelecer retorno sobre o investimento esperado.

**Esses requisitos fornecem o contexto necessário para que as decisões técnicas sejam alinhadas com as reais necessidades do negócio.**
