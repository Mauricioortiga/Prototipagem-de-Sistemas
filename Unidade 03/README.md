A construção de sistemas computacionais eficientes e alinhados com as necessidades dos usuários depende fundamentalmente de uma 
representação clara e compartilhada de como o sistema deve se estruturar e comportar-se.

**Essa realidade prática nos leva a uma necessidade essencial: a modelagem estrutural e comportamental de sistemas.**

A modelagem estrutural:
**Nos ajuda a entender como o sistema é organizado:** 

– Quais são seus componentes.

- Como se relacionam e como se integram.

A modelagem comportamental:

**Nos permite visualizar o que o sistema faz:

- Como os objetos interagem.

-Quais são os fluxos de processos.

-Como as decisões são tomadas.

**Essas perspectivas formam a base para o desenvolvimento mais consciente e alinhado com as necessidades reais dos projetos, permitindo 
uma comunicação clara entre stakeholders técnicos e não técnicos, além de facilitarem a identificação precoce de problemas e redundâncias no design.**

Objetivos

- Compreender as perspectivas complementares das modelagens estrutural (organização) e comportamental (ação) para o sucesso de projetos de software;

- Aplicar o modelo C4 para comunicar a arquitetura do sistema em diferentes níveis de abstração, do contexto geral ao código;

- Dominar diagramas de classe para representar a estrutura estática, incluindo atributos, métodos e tipos de relacionamentos entre objetos;

- Utilizar diagramas de sequência para documentar as interações dinâmicas e a troca de mensagens entre componentes ao longo do tempo;

- Aplicar diagramas de atividades para mapear fluxos de processos, decisões lógicas e responsabilidades em sistemas complexos.
