1. A Modelagem C4 é projetada para diferentes níveis de abstração, permitindo que stakeholders técnicos e não técnicos compreendam a arquitetura do sistema em níveis adequados às suas necessidades. Os diagramas de classe e sequência são mais técnicos, enquanto os diagramas de atividades focam em fluxos de processos.

2. A herança (ou generalização) expressa um relacionamento "é um", ou seja, é um tipo específico, onde uma classe filha (Revista) herda características de uma classe pai (Periódico). A revista é um tipo específico de periódico.

3. Diagramas de atividades são ideais para modelar fluxos de controle, decisões e processos complexos. O uso de raias (raias) permite separar as responsabilidades de cada ator no processo.

4. A visibilidade de um atributo (+ para público, - para privado, # para protegido) indica quem pode acessar esse atributo.

5. A herança permite reutilizar código e estabelecer hierarquias lógicas, onde classes filhas (eletrônicos, livros, roupas) herdam atributos e comportamentos de uma classe pai (Produto).

6. O Diagrama de Contêineres representa as unidades de implantação separadas que compõem o sistema, como aplicações web, bancos de dados e servidores de API.

7. Diagramas de sequência mostram como os objetos interagem ao longo do tempo para realizar uma tarefa, sendo ideais para documentar fluxos complexos e interações entre componentes.

8. Diagramas de sequência mostram como os objetos interagem ao longo do tempo, representando a troca de mensagens (chamadas de método, requisições) entre os participantes.
