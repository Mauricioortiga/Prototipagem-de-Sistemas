🔍**Quando falamos de levantamento de requisitos, precisamos:**
1. Descobrir
2. Analisar
3. Documentar
4. Validar as necessidades dos **stakeholders** (O contratante que tem interesse ou é afetado por um projeto ou sistema)

Estes requisitos são descrições do que o sistema deve fazer, incluindo serviços e limitações. 
Para criar protótipos eficazes, é importante usar diferentes técnicas que capturem tanto as necessidades claras quanto as escondidas dos usuários.

---
🔨 **Brainstorming**

Destaca-se como uma das principais técnicas de levantamento de requisitos:

É uma abordagem **colaborativa** que incentiva a troca de ideias em grupo. **Stakeholders e equipe compartilham sugestões e necessidades sem julgamentos**, 
ajudando a identificar requisitos inovadores para explorar várias soluções, especialmente nas fases iniciais do projeto.

---
🔨 **Entrevistas e Questionários**

**Entrevista:**

Técnica fundamental para coleta de requisitos, permite a interação direta com Stakeholders.

A entrevista consiste em dois tipos principais:

1. Realizar entrevista estruturada, que segue um roteiro pré definido de perguntas.

2. Realizar entrevista não estruturada, permitindo explorar o problema de forma mais livre.

🎈 Page-Jones (2001) recomenda:

1. Preparar-se adequadamente, estudando o domínio do problema

2. Combinar perguntas abertas e fechadas

3. Validar entendimentos durante a conversa

4. Documentar insights (compreensão) rapidamente após cada sessão.

**Questionário:**

Questionários permitem coletar dados de muitos stakeholders de forma padronizada. 

São úteis para validar requisitos já identificados e obter feedback de forma quantitativa, sendo APROPRIADA para complemento de entrevistas e observações diretas.

---
🔨 **Observação e Análise de Dados do Usuário:**

Observar diretamente os usuários em seu ambiente de trabalho ajuda a identificar necessidades que eles podem não conseguir explicar.

🎈 Pressman (2021) destaca que muitos usuários não conseguem explicar detalhadamente suas tarefas diárias, por isso a observação é útil para identificar 
processos e necessidades que não apareceriam apenas em entrevistas ou questionários.
